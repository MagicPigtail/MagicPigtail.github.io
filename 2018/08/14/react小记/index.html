react小记

发表于 2018-08-14

这篇是对于利用react、redux、dva、antd Pro等开发的经验总结。为了在以后遇到各种各样的问题时可以回头看看，借鉴一下。

React

react检测不到数据改变

react是通过diff算法和虚拟dom来减少对真实dom的操作，从而提高渲染效率，当视图层依赖的数据发生改变时，通过diff算法，将调整页面的事件放在一个队列中，当所有数据改变计算完成后，执行事件队列中的事件，改变虚拟dom，而实际上对真实dom的渲染只限于依赖数据改变的地方。但在有些情况下react检测不到数据的改变。

例如当一个试图依赖的数据是一个存在于JSON对象中嵌套较深的一个属性时，而且这个数据是个数组，当我们对这个数组进行浅拷贝(如push,shift等方法)改变它的值时，页面不会触发重新渲染，原因是该Array对象是一个引用类型，浅拷贝不会改变其指针，当嵌套较深时指针不变react默认其值未发生变化，若想要触发重新渲染，需要对其值进行深拷贝，调用slice,concat方法或重新新建一个对象将原始对象进行赋值传递属性给它即可。

不管时存储在model中全局store里的数据还是组件内部的state中的数据，react对数据检测的方案是一样的，因为store本质上即一个顶层组件的state，但在model中对其的改变都存在于reducer中，我们是通过声明一个新的state，然后将老的state通过扩展符{...state}的方式深拷贝给新的state，而在组件内部的state里我们应该用setState的方法对其赋值而不是通过显示赋值。

运用箭头函数

react的每一个组件都是一个类，而render出来的html需要绑定该类中的一些方法，这些方法有时候会用到全局store中的值或者要dispatch一个action，这个时候必须要拿到当前对象即this，如果直接用函数声明的方法，则必须要再constructor里将每个函数用bind方法改变作用域，但如果用箭头函数的函数表达式方法声明，因为箭头函数的this始终是函数定义时绑定，所以可以避免在constructor里依次bind的冗余操作。

这里的箭头函数的this在定义时绑定指的是箭头函数的this永远指向父级的上下文。而普通函数的this指向的时调用它的对象。

setState()是异步的

this.setState()会调用render方法，但并不会立即改变state的值，state是在render方法中赋值的。所以执行this.setState()后立即获取state的值是不变的。同样的直接赋值state并不会触发更新，因为没有调用render函数。

Redux

数据存放方案

全局store的存在给我们处理组件间通信提供了便捷，但我们不应该把所有数据放在它里面，原因是我们应该尽可能使组件间解耦。
当一个页面功能为纯展示页面，且依赖的数据量不是非常庞大时，我们可以考虑将该页面的数据存储在全局store中，如此一来，当页面路由跳转时，该数据不会被清理掉，当其他页面需要这些数据时不用再向后台请求，可以直接在store中拿到。

总结：以下几种情况可以考虑存储在全局store中

  1. 当一个数据同时被几个组件所依赖
  2. 从后台请求过来的数据
  3. 该数据用来纯展示，但展示条件会经常变化，且数据量不是特别庞大，变化较稳定
